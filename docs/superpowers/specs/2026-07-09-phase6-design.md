## 1. Task 26: Zero-Drop Guarantee (零丢失保障机制)

### 1.1 Problem

`filter_rows_by_evidence()` in `core/pipeline.py` uses post-hoc fuzzy title matching (`_find_matching_paper`) to link LLM-extracted rows back to source papers. When `pdf_parser.py` cannot extract a title (non-standard PDF → `paper.title == "missing"`), the matcher fails silently. The row is blocked and never enters the final artifacts (`survey_draft.tex`, `matrix_table.tex`, `references.bib`).

**Result:** N PDFs in → M < N rows out. Papers vanish without error.

### 1.2 Solution: Inline Scope Binding

Replace the flat-list `filter_rows_by_evidence()` with per-paper inline validation inside the main `for paper in papers` loop.

**Old pipeline:**
```
for paper in papers:
    rows = extract_with_self_healing(paper)
    all_rows += rows
accepted, blocked = filter_rows_by_evidence(all_rows, papers)   ← BUG: fuzzy match fails
generate_artifacts(accepted)                                     ← MISSING papers
```

**New pipeline:**
```
for paper in papers:
    rows = extract_with_self_healing(paper)
    for row in rows:
        result = validate_evidence(row, paper.page_text_by_number())  ← DIRECT binding
        if result.accepted or max_retries_exhausted:
            all_rows.append(row)                                      ← ALWAYS appended
generate_artifacts(all_rows)                                          ← N rows guaranteed
```

### 1.3 File Changes

| File | Change |
|------|--------|
| `core/pipeline.py` | Remove `filter_rows_by_evidence()`, `_find_matching_paper()`. Refactor `extract_with_self_healing()` to accept optional `paper` parameter for inline binding. |
| `scripts/run_extraction.py` | Remove call to `filter_rows_by_evidence()`; use inline binding in main loop. |
| `main.py` | Same removal as `run_extraction.py`. |
| `tests/test_pipeline.py` | Replace `test_filter_rows_by_evidence_*` tests with Zero-Drop inline binding tests. |

### 1.4 Acceptance Criteria

- Input N PDFs → exactly N rows in final artifacts (excluding `paper.error` ones).
- Rows whose evidence fails containment are force-included with `"missing (unverified)"` status.
- No paper is ever silently dropped.

## 2. Task 24: 50,000-Character Word Count & Max Output Token Defense

### 2.1 Problem

LLMs have a hard physical limit on max output tokens (~4096-8192 tokens ≈ 3000-6000 Chinese chars). A single-pass API call for a 50,000-char manuscript will be truncated mid-sentence, producing LaTeX without `\end{document}`.

### 2.2 Solution: Hybrid Strategy

**Threshold gate:** ≤8000 chars → single-pass (current logic, backward compatible).  
**>8000 chars → multi-stage synthesis:** Split the 6 `\section{...}` into 6 independent LLM calls.

### 2.3 Multi-stage Synthesis Flow: Chained Contextual Generation

```
word_count_target > 8000 ?
  No  → render_survey_tex_with_llm()       (single-pass, existing logic)
  Yes → render_survey_tex_multi_stage()     (new function)
           ├── build_preamble()             [hardcoded, includes % !TEX magic comments]
           ├── Stage 1: rows + section-1 prompt  →  Section 1 content
           ├── Stage 2: rows + Sections 1-2 prompt + Section 1 real text  →  Section 2
           ├── Stage 3: rows + Sections 1-3 prompt + Sections 1-2 real text  →  Section 3
           ├── Stage 4: rows + Sections 1-4 prompt + Sections 1-3 real text  →  Section 4
           ├── Stage 5: rows + Sections 1-5 prompt + Sections 1-4 real text  →  Section 5
           ├── Stage 6: rows + prompt + Sections 1-5 real text  →  Section 6
           └── append \end{document}         [hardcoded]
```

Each section call receives:
1. Full rows matrix JSON (all papers, all fields) — ensures cross-paper comparison.
2. Previously generated sections 1..N-1 as chained context — ensures style consistency and logical flow.
3. Section-scoped prompt — limits generation to that section's content only.

**Preamble strategy (Method B):** Hardcoded in Python, never generated by LLM:
```latex
% !TEX program = xelatex
% !TEX root = survey_draft.tex
\documentclass{ctexart}
\usepackage{booktabs}
\usepackage{tabularx}
\usepackage[backend=biber,style=gb7714-2015]{biblatex}
\addbibresource{ref.bib}
\begin{document}
```

**Ending strategy (Method D):** `\end{document}` appended by Python code after Section 6.

### 2.4 File Changes

| File | Change |
|------|--------|
| `main.py` | Slider range: 1000–50000, step 500. Add threshold warning text. |
| `core/synthesis.py` | Add `render_survey_tex_multi_stage()`, `_build_section_prompt()`. Modify `render_survey_tex_with_llm()` to dispatch. |
| `core/pipeline.py` | `generate_llm_artifacts()` passes `word_count_target` through; synthesis module handles dispatch internally. |

### 2.5 Acceptance Criteria

- Slider range 1000–50000.
- ≤8000 → single-pass (backward compatible).
- >8000 → multi-stage, each section generated independently.
- Final stitched LaTeX passes `validate_latex_syntax()`.
